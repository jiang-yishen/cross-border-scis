## 二、跨境电商海外仓行业KPI基准数据

### 1. 库存管理核心KPI（行业通用）

| KPI指标 | 计算公式 | 行业基准 | 优秀水平 | 头部企业 |
|---------|----------|----------|----------|----------|
| **库存周转率** | 年销售成本 / 平均库存 | 4-6次/年 | 6-8次/年 | 10-12次/年 |
| **库存周转天数** | 平均库存 / 日均销售成本 | 60-90天 | 45-60天 | 30-45天 |
| **库存持有成本率** | 持有成本 / 平均库存金额 | 15-20% | 8-12% | <8% |
| **缺货率（Stockout Rate）** | 缺货次数 / 总需求次数 | 5-15% | 3-5% | <2% |
| **服务水平（Service Level）** | 订单完全满足率 | 90-95% | 95-97% | 97-99% |
| **订单满足率（Fill Rate）** | 已满足数量 / 订单数量 | 92-95% | 95-97% | ≥98% |
| **库存准确率** | 1 - \|盘点差异\|/账面库存 | 85-95% | 95-98% | >99% |
| **预测准确率（Forecast Accuracy）** | 100% - MAPE | 60-75% | 80-90% | >90% |
| **滞销品占比** | 滞销品金额 / 总库存金额 | 15-25% | 10-15% | <10% |
| **库存售罄率** | 90天销售 / 90天收货 | 60-80% | 80%+ | 目标80%+ |

### 2. 跨境电商海外仓专项KPI

| KPI指标 | 行业基准 | 说明 |
|---------|----------|------|
| **仓储运营成本率** | 占出库货值的3-8% | 含租金、人工、水电、设备折旧 |
| **跨境物流单均成本** | 轻小件≤$5，大件≤$15 | 海外仓到终端客户的配送成本 |
| **库存周转天数（快消）** | ≤45天 | 快消品、美妆等 |
| **库存周转天数（3C）** | ≤60天 | 电子、家居等 |
| **库存周转天数（服装）** | ≤30-45天 | 季节性强的品类 |
| **跨仓调拨效率** | 调拨周期3-7天 | 多仓协同的关键指标 |
| **退货处理时效** | 48-72小时 | 退货检测、换标、重新上架 |
| **FBA费用占比** | 占销售额15-30% | 含仓储、配送、退货处理 |

## 四、技术栈与算法模型推荐

### 1. 需求预测算法

| 算法 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| **Prophet** | 长期趋势+季节性+节假日 | 可解释性强，业务人员友好 | 对短期波动不敏感 |
| **XGBoost/LightGBM** | 短期预测，多特征融合 | 准确率高，可处理非线性关系 | 需要特征工程 |
| **ARIMA/SARIMA** | 平稳时间序列 | 数学基础扎实 | 对复杂模式拟合弱 |
| **LSTM/Transformer** | 大规模数据，复杂模式 | 捕捉长期依赖关系 | 训练成本高，黑盒 |
| **集成学习（Ensemble）** | 综合场景 | 综合各模型优势，鲁棒性高 | 维护成本高 |

> **推荐组合**：**Prophet（长期趋势+节假日） + XGBoost（短期+多特征）**，这是目前工业界最成熟、性价比最高的组合。

### 2. 库存优化算法

| 算法/模型 | 用途 | 公式/说明 |
|-----------|------|-----------|
| **EOQ（经济订货量）** | 最优订货批量 | √(2 × D × S / H) |
| **动态安全库存** | 应对需求波动 | SS = Z × σ_d × √(LT) |
| **ROP（再订货点）** | 触发补货时机 | ROP = (日均需求 × LT) + SS |
| **ABC-XYZ分类** | SKU分级管理 | ABC（价值贡献）+ XYZ（需求稳定性） |
| **蒙特卡洛模拟** | 风险评估 | 模拟多种需求场景下的库存表现 |
| **OR-Tools** | 路径/分配优化 | Google开源运筹学工具 |

### 3. Web应用技术栈（可交互Demo）

| 层级 | 推荐方案 | 说明 |
|------|----------|------|
| **后端** | FastAPI / Flask | Python生态，ML模型部署友好 |
| **前端** | React + Tailwind CSS | 现代化UI，响应式，适合手机端 |
| **可视化** | Plotly / ECharts | 交互式图表，支持缩放、筛选 |
| **数据** | Pandas / SQLite | 轻量级，适合Demo演示 |
| **部署** | Streamlit / Vercel | 快速部署，支持手机浏览器访问 |

> **最简可行方案**：**Streamlit**（纯Python）或 **FastAPI+React**（前后端分离）。如果追求开发速度和视觉效果，推荐 **FastAPI + React + Plotly**。

## 五、建议你项目聚焦的优化环节与KPI

基于以上调研，结合你的业务规模（1000-1200 SKU，4-5个海外仓），我推荐以下**3个核心优化环节**和**对应的KPI目标**：

### 核心环节1：需求预测优化
| 优化前 | 优化后 | 量化目标 |
|--------|--------|----------|
| 预测准确率 ~60% | 预测准确率 **>85%** | 提升 **40%+** |
| 人工Excel预估 | Prophet+XGBoost自动化预测 | 预测效率提升 **10倍** |
| 无季节性建模 | 融合节假日/促销/季节性因子 | 大促期间预测偏差 **<15%** |

### 核心环节2：智能补货与库存优化
| 优化前 | 优化后 | 量化目标 |
|--------|--------|----------|
| 库存周转率 ~4次/年 | 库存周转率 **6-8次/年** | 提升 **50-100%** |
| 缺货率 10-15% | 缺货率 **<5%** | 降低 **60%+** |
| 安全库存一刀切 | 动态安全库存（ABC-XYZ分级） | 安全库存资金占用降低 **20-30%** |
| 库存周转天数 60-90天 | 库存周转天数 **45-60天** | 缩短 **25-50%** |

### 核心环节3：采购计划与物流跟踪
| 优化前 | 优化后 | 量化目标 |
|--------|--------|----------|
| 采购计划无跟踪 | 采购全流程可视化（PO→收货→质检） | 采购到货准时率 **>90%** |
| 物流异常被动发现 | 物流预警系统（在途异常自动提醒） | 异常响应时间从 **3天→1天** |
| 无供应商绩效评估 | 供应商KPI看板（交期/质量/成本） | 供应商准时交付率 **>95%** |
